# Claude Code 研究工作者使用指南

> 专为计算机科研工作者设计的 Claude Code 使用手册

---

## 一、Claude Code 概述

Claude Code 是 Anthropic 官方推出的命令行 AI 助手，通过自然语言与代码交互，是科研人员提升编码效率的利器。

**核心优势：**
- 深度代码理解与生成
- 上下文持续跟踪
- 无缝集成开发环境（IDE）
- 可扩展的 Agent Skills

---

## 二、核心斜杠命令 (Slash Commands)

### 2.1 基础命令

| 命令 | 说明 | 科研场景示例 |
|------|------|--------------|
| `/help` | 显示所有可用命令 | 遇到问题时快速查阅 |
| `/clear` | 清除对话上下文 | 切换实验分支时重新开始 |
| `/exit` | 退出 Claude Code | 完成工作后退出 |

### 2.2 记忆管理

| 命令 | 说明 | 科研场景示例 |
|------|------|--------------|
| `/remember <内容>` | 记住配置或偏好 | 记住项目代码风格、实验参数 |
| `/forget` | 清除记忆 | 实验配置变更后重置 |

**科研示例：**
```
/remember 本项目使用 PEP8 代码风格，所有函数需添加类型注解
```

### 2.3 任务管理

| 命令 | 说明 |
|------|------|
| `/tasks` | 查看当前任务列表 |
| `/tasks -a` | 查看所有任务（含已完成） |

## 三、Agent Skills 详解

### 3.1 内置 Skills

| Skill | 功能 | 科研应用场景 |
|-------|------|--------------|
| `commit` | 智能 Git 提交 | 提交实验代码、版本管理 |
| `review-pr` | 审查 Pull Request | 代码审查、论文代码检查 |
| `pdf` | 处理 PDF 文件 | 分析论文、提取公式、解析 API 文档 |

### 3.2 调用 Skills 的两种方式

**方式一：直接斜杠命令**
```bash
/commit -m "实现 ResNet 变体"
/review-pr 42
/pdf /path/to/paper.pdf
```

**方式二：自然语言触发**
```
帮我分析这篇论文 /pdf /path/to/paper.pdf
```

**⚠️ 注意：** 只有看到 `<command-name>` 标签时，Skill 才已成功加载。

## 四、科研工作流实战

### 4.1 论文代码复现

```bash
# 步骤 1：让 Claude 理解论文
请帮我阅读 /pdf ./papers/attention.pdf 并总结核心算法

# 步骤 2：生成代码框架
基于上述论文的 Algorithm 1，用 PyTorch 实现核心模块

# 步骤 3：调试与优化
我的实现与论文 Figure 3 结果不一致，帮我分析原因

# 步骤 4：代码整理与提交
整理实验代码，添加注释，然后 /commit
```

### 4.2 实验管理

```bash
# 查看当前实验目录结构
ls experiments/

# 为新实验创建结构化目录
创建实验 exp_2026_01_26_transformer，包含 models/, data/, results/, logs/

# 实现实验代码
实现 Transformer 模型，添加实验日志记录功能

# 运行实验
python train.py --config configs/exp_26.yaml

# 记录实验结果
实验 accuracy 达到 89.5%，对比 baseline 提升了 3.2%，帮我更新 README
```

### 4.3 数据处理与分析

```bash
# 数据集分析
分析 ./data/dataset.csv 的统计特征，识别异常值

# 数据预处理
设计数据预处理 pipeline，包括归一化、缺失值填充

# 可视化
绘制训练曲线和混淆矩阵，保存到 ./figures/

# 代码优化
当前数据加载是瓶颈，请帮我优化 dataloader 性能
```

### 4.4 代码审查与合作

```bash
# 自我审查
审查我最近的代码变更，检查是否有潜在 bug

# 团队合作场景
/review-pr 123
```

## 五、命令与 Skills 的配合模式

### 5.1 模式对照表

| 模式 | 适用场景 | 示例 |
|------|----------|------|
| **先分析后 Skill** | 需要先理解代码状态 | `git diff` → `/commit` |
| **直接调用 Skill** | 明确知道要做什么 | `/commit -m "..."` |
| **自然语言驱动** | 不确定具体步骤 | "帮我优化这个模型" |
| **组合工作流** | 复杂多步任务 | "实现算法并生成报告" |

### 5.2 科研典型工作流

**工作流 A：阅读论文 → 实现算法 → 实验验证 → 提交代码**
```
1. /pdf ./papers/new_method.pdf
2. 请总结核心贡献和算法流程
3. 基于第 3 节的算法，用 PyTorch 实现
4. 设计对照实验
5. 运行实验并分析结果
6. /commit -m "实现新方法论文算法"
```

**工作流 B：调试实验 → 修复代码 → 更新文档**
```
1. 实验报错，帮我分析 log.txt
2. 修复这个 bug
3. 重新运行验证
4. 更新实验文档记录修复过程
```

## 六、高级科研技巧

### 6.1 记忆科研约定

```bash
/remember
- 本项目使用 PyTorch 2.0+
- 所有实验结果保存在 ./results/
- 代码遵循 Google Python 风格指南
- 实验配置使用 YAML 文件管理
```

### 6.2 并行操作

在单条消息中让 Claude 同时执行多个独立任务：
```
同时执行：
1. 检查代码风格
2. 运行单元测试
3. 生成实验报告
```

### 6.3 批量处理

```
批量处理 ./data/raw/ 下的所有 CSV 文件，生成统计报告
```

### 6.4 实验追踪

```
帮我对比 exp_1、exp_2、exp_3 三个实验的结果，生成对比表格
```

## 八、最佳实践

1. **先描述，后执行** - 清晰描述实验目标，让 Claude 理解上下文
2. **善用 /remember** - 记住项目约定，避免重复说明
3. **分步验证** - 复杂实验分步执行，每步验证结果
4. **保留实验记录** - 代码提交时附带有意义的实验信息
5. **定期整理** - 用 `/tasks` 跟踪进度，完成后清理
