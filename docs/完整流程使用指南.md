# 视频轨迹对齐工具 - 完整流程使用指南

## 📋 概述

这是一个重新设计的数据处理工具,将整个流程分为 4 个清晰的步骤:

### 流程图

```
视频文件 (.mp4)
    ↓
步骤1: 提取帧 (每秒一帧)
    ↓
步骤2: OCR识别 + 重命名
    ↓
步骤3: 删除重复帧
    ↓
步骤4: 轨迹数据对齐
    ↓
最终数据 (CSV + JSON + 帧图像)
```

---

## 🚀 快速开始

### 基本用法


## 📖 详细步骤说明

### 步骤1: 提取视频帧

**类**: `VideoFrameExtractor`

**功能**:
- 扫描视频文件夹中的所有 `.mp4` 文件
- 从每个视频中每秒提取一帧
- 保存为临时文件 (`temp_VIDEO_SECOND.jpg`)

**输入**:
- `data/video/B-2024-10-18/` - 视频文件夹

**输出**:
- `data/aligned_output/extracted_frames/temp_*.jpg` - 临时帧文件

**示例输出**:
```

================================================================================

  时长: 1797 秒, FPS: 24.00
  提取帧: 100%|████████| 1797/1797

✓ 总共提取了 25185 个帧
```

---

### 步骤2: OCR识别和重命名

**类**: `FrameTimeRecognizer`

**功能**:
- 对每个临时帧进行 OCR 识别
- 提取时间戳
- 重命名为 `YYYYMMDD_HHMMSS.jpg` 格式
- 如果重名,添加序号后缀 `_001`

**OCR 策略**:
- 尝试不同的图像区域大小
- 尝试不同的 OCR 配置参数
- 支持多种时间格式

**输入**:
- `data/aligned_output/extracted_frames/temp_*.jpg`

**输出**:
- `data/aligned_output/extracted_frames/YYYYMMDD_HHMMSS.jpg` (重命名后的帧)

**示例输出**:
```

================================================================================
✓ 找到 25185 个临时帧文件

OCR识别: 100%|████████| 25185/25185

✓ OCR成功: 24000
✓ OCR失败: 1185
```

---

### 步骤3: 删除重复帧

**类**: `FrameDeduplicator`

**功能**:
- 识别具有相同时间戳的帧
- 保留第一个,删除后续的重复帧
- 避免数据冗余

**输入**:
- `data/aligned_output/extracted_frames/YYYYMMDD_HHMMSS.jpg`

**输出**:
- 删除重复后的帧文件

**示例输出**:
```
步骤3: 删除重复帧
================================================================================
✓ 删除了 515 个重复帧
✓ 剩余帧数: 23485
```

---

### 步骤4: 轨迹数据对齐

**类**: `TrajectoryAligner`

**功能**:
- 加载轨迹数据 (Excel)
- 为每个帧查找匹配的轨迹记录
- 在时间容差范围内进行匹配
- 复制帧到最终目录
- 生成对齐后的数据文件

**输入**:
- `data/trajectory/B-2024-10-18/12-12-49_23-59-58.xlsx` - 轨迹数据
- `data/aligned_output/extracted_frames/*.jpg` - 去重后的帧

**输出**:
- `data/aligned_output/aligned_frames/` - 最终帧目录
- `data/aligned_output/aligned_data.csv` - 对齐数据 (CSV)
- `data/aligned_output/aligned_data.json` - 对齐数据 (JSON)
- `data/aligned_output/alignment_stats.json` - 统计信息

**示例输出**:

## ⚠️ 注意事项

### 1. 磁盘空间

- **临时帧文件**: 可能占用大量空间
- **最终帧文件**: 约 2-3 GB
- **建议**: 确保至少 10 GB 可用空间

### 2. 处理时间

- **步骤1** (提取帧): 5-10 分钟
- **步骤2** (OCR): 30-60 分钟 (取决于帧数)
- **步骤3** (去重): 1-2 分钟
- **步骤4** (对齐): 2-5 分钟

### 3. OCR 成功率

- **典型成功率**: 90-95%
- **失败处理**: 删除临时文件,不保留
- **影响因素**: 图像质量、光照、分辨率

### 4. 时间容差

- **默认**: 2 秒
- **建议**: 根据轨迹数据采样率调整
- **过大**: 可能导致错误匹配
- **过小**: 可能导致丢失数据

## 🐛 常见问题

### Q1: OCR 失败率高怎么办?

A: 尝试以下方法:
1. 检查视频质量
2. 调整 OCR 区域参数
3. 增加二值化阈值
4. 使用其他 OCR 引擎

### Q2: 帧数量少于预期?

A: 可能原因:
1. 视频时长不足
2. OCR 失败率高
3. 重复帧过多
4. 时间容差设置不当

### Q3: 如何验证数据质量?

A: 检查:
```bash
# 检查文件数量
ls data/aligned_output/aligned_frames/ | wc -l
wc -l data/aligned_output/aligned_data.csv

# 检查时间范围
head -20 data/aligned_output/aligned_data.csv
tail -20 data/aligned_output/aligned_data.csv
```

### Q4: 可以跳过某些步骤吗?

A: 可以,但需要:
1. 确保输入文件格式正确
2. 手动管理中间文件
3. 了解每个步骤的依赖关系

## 🎯 与旧脚本的对比

### 旧流程 (`align_video_trajectory_final.py`)

```
视频 → 计算时间 → 提取帧 → 对齐
        ↑
    (可能不准确)
```

### 新流程 (`align_complete_pipeline.py`)

```
视频 → 提取帧 → OCR识别时间 → 去重 → 对齐
                  ↑
            (更准确)
```

### 优势

1. **更准确**: 使用 OCR 实际识别时间,而不是推算
2. **更清晰**: 每个步骤职责明确
3. **更可靠**: 有去重步骤,避免数据冗余
4. **更灵活**: 可以单独运行每个步骤
5. **更易维护**: 代码模块化,易于调试
