## 🔧 技术细节

### 时间戳识别

#### 问题发现
- 文件名时间与实际视频时间存在**8小时偏移**
- 文件名: `04:38:10` → 实际: `12:38:12`
- 文件名: `10:11:29` → 实际: `18:11:31`

#### 解决方案
使用OCR从视频帧识别实际时间戳：
- 从最后一帧提取结束时间（识别率最高）
- 根据视频时长反推开始时间
- 准确率: 100%（4/4个视频）

### 时区问题修复

#### 问题描述
- `datetime.timestamp()` 返回本地时区时间戳
- `pd.Timestamp().timestamp()` 返回UTC时间戳
- 差异: 8小时 (28,800秒)

#### 解决方案
统一使用 `pd.Timestamp(dt).timestamp()` 进行时间戳转换

## 🚀 使用方法

### 查看对齐数据

```bash
# CSV格式
head data/aligned_output/aligned_data.csv

# JSON格式
python3 -c "import json; data = json.load(open('data/aligned_output/aligned_data.json')); print(data[0])"
```

### 加载对齐数据

```python
import pandas as pd

# 加载CSV
df = pd.read_csv('data/aligned_output/aligned_data.csv')

# 查看统计
print(f"总帧数: {len(df)}")
print(f"视频文件: {df['video_file'].unique()}")
print(f"时间范围: {df['frame_time'].min()} 到 {df['frame_time'].max()}")

# 按视频分组
for video, group in df.groupby('video_file'):
    print(f"{video}: {len(group)} 帧")
```

### 访问帧图像

```python
from PIL import Image
import pandas as pd

df = pd.read_csv('data/aligned_output/aligned_data.csv')

# 获取第一帧
first_frame_path = df.iloc[0]['frame_path']
image = Image.open(first_frame_path)

# 显示帧信息
print(f"图像大小: {image.size}")
print(f"时间: {df.iloc[0]['frame_time']}")
print(f"位置: ({df.iloc[0]['经度']}, {df.iloc[0]['纬度']})")
print(f"速度: {df.iloc[0]['速度']} m/s")
```
