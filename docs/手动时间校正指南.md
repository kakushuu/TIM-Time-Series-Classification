# 手动时间校正指南

## 📋 问题描述

在 `20241018_184134` (18:41:34) 之后的帧，时间戳可能不准确：
- 有的比实际时间少1秒
- 有的比实际时间多2秒

## 🔍 解决方案

### 方案1：使用交互式工具（推荐）

```bash
# 启动交互式调整工具
python3 scripts/manual_time_adjustment.py --interactive
```

然后按照提示操作：
1. 输入视频文件名
2. 输入需要调整的秒数范围
3. 输入时间偏移量
4. 重复直到完成

### 方案2：使用配置文件

#### 步骤1: 创建调整配置模板

```bash
python3 scripts/manual_time_adjustment.py --create-template
```

这将创建 `time_adjustments_template.json` 文件。

#### 步骤2: 编辑配置文件

打开 `time_adjustments_template.json`，修改 `adjustments` 数组：

```json
{
  "adjustments": [
    {
      "video_file": "20241018104130.mp4",
      "start_second": 0,
      "end_second": 100,
      "offset_seconds": 1,
      "reason": "前100秒需要加1秒"
    },
    {
      "video_file": "20241018111131.mp4",
      "start_second": 0,
      "end_second": 100,
      "offset_seconds": -2,
      "reason": "前100秒需要减2秒"
    }
  ]
}
```

#### 步骤3: 应用调整

```bash
python3 scripts/manual_time_adjustment.py --apply time_adjustments_template.json
```

### 方案3：直接告诉我调整参数


## 🔧 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `video_file` | 视频文件名 | `20241018104130.mp4` |
| `start_second` | 起始秒数（second_in_video） | `0` |
| `end_second` | 结束秒数（second_in_video） | `100` |
| `offset_seconds` | 时间偏移（正数=加，负数=减） | `1` 或 `-2` |
| `reason` | 调整原因（可选） | `"前100秒时间不准" |

## 📝 注意事项

1. **备份原始数据**: 调整会修改CSV文件，建议先备份
2. **调整范围**: `start_second` 和 `end_second` 是 `second_in_video` 字段的值
3. **偏移方向**: 正数=时间延后，负数=时间提前
4. **覆盖**: 如果多次调整同一帧，后执行的会覆盖前面的
5. **时间连续性**: 调整后可能导致时间不连续，请检查
