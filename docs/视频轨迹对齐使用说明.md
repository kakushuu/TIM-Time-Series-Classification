# 视频与轨迹数据对齐工具使用说明

## 📋 概述

本工具用于将视频数据与轨迹数据按时间戳对齐，提取每秒一帧并与轨迹记录匹配。

## ⚠️ 当前数据问题

### 时间范围分析

**视频数据（B-2024-10-18）**：
- 时间范围：2024-10-18 04:38:10 到 11:41:28
- 总时长：约6.5小时
- 视频数量：16个MP4文件

**轨迹数据（12-12-49_23-59-58.xlsx）**：
- 时间范围：2024-10-18 12:12:49 到 23:59:58
- 总时长：约12小时
- 记录数量：42,430条

**问题**：这两个数据集的时间**完全不重叠**！
- 视频数据在轨迹数据之前约31分钟
- 无法进行时间对齐

## 🔍 可能的原因

1. **缺失视频文件**：可能有11:41之后的视频文件未上传
2. **错误的轨迹文件**：可能使用了错误日期的轨迹数据
3. **文件命名错误**：轨迹文件名显示时间范围（12-12-49_23-59-58）与内容时间不匹配

## 🛠️ 解决方案

### 方案1：获取完整数据

检查是否有其他时间段的数据：
- 11:41到12:12之间的视频（约31分钟空白）
- 12:12之后的视频文件

### 方案2：使用正确的数据文件

如果有其他日期或时间段的数据，请使用正确的文件路径：

```bash
python3 scripts/align_video_trajectory_v2.py \
  --trajectory data/trajectory/<正确的日期>/<正确的文件>.xlsx \
  --video-dir data/video/<正确的日期> \
  --output data/aligned_output
```

### 方案3：测试脚本（使用现有数据的一部分）

如果只是想测试脚本功能，可以创建测试数据：


## 📝 脚本说明

### 主脚本：`align_video_trajectory_v2.py`

**功能**：
1. 自动检测视频和轨迹数据的时间范围
2. 检查时间重叠并给出警告
3. 使用视频文件名中的时间戳（更可靠）
4. 每秒提取一帧并与轨迹数据对齐
5. 支持时间容差匹配（默认2秒）
6. 生成详细的对齐报告

**参数**：
- `--trajectory, -t`：轨迹数据Excel文件路径
- `--video-dir, -v`：视频文件夹路径
- `--output, -o`：输出目录
- `--use-ocr`：使用OCR验证时间戳（实验性）
- `--time-tolerance`：时间容差（秒），默认2秒

**输出**：
- `aligned_data.csv`：对齐后的数据（CSV格式）
- `aligned_data.json`：对齐后的数据（JSON格式）
- `alignment_stats.json`：对齐统计信息
- `aligned_frames/`：提取的视频帧目录
  - 帧文件命名：`YYYYMMDD_HHMMSS.jpg`

### 辅助脚本

1. **`test_extract_frames.py`**：提取视频的第一帧和最后一帧
2. **`test_ocr.py`**：测试OCR时间戳识别
3. **`analyze_video_timestamps.py`**：分析所有视频的时间范围

## 🚀 使用示例

### 正常使用（当数据时间匹配时）

```bash
python3 scripts/align_video_trajectory_v2.py \
  --trajectory data/trajectory/2024-10-18/data.xlsx \
  --video-dir data/video/2024-10-18 \
  --output data/output
```

### 带参数使用

```bash
python3 scripts/align_video_trajectory_v2.py \
  -t data/trajectory/data.xlsx \
  -v data/video/ \
  -o data/output \
  --time-tolerance 5
```

## ⚙️ 技术细节

### 时间戳提取

1. **主要方法**：从视频文件名解析
   - 格式：`YYYYMMDDHHMMSS.mp4`
   - 例如：`20241018043810.mp4` → 2024-10-18 04:38:10
   - 准确率：100%

2. **辅助方法**：OCR识别（可选）
   - 从视频帧左上角识别时间戳
   - 适用于验证或文件名无时间戳的情况
   - 准确率：约70-80%（受图像质量影响）

### 时间对齐算法

1. 将轨迹数据时间转换为Unix时间戳
2. 创建时间戳到数据行的映射表
3. 对每一帧计算对应时间
4. 在时间容差范围内查找匹配的轨迹数据
5. 保存匹配的帧和轨迹数据

### 性能优化

- 使用时间戳索引快速查找
- 批量处理视频文件
- 支持时间容差匹配（处理时间漂移）

## 🐛 常见问题

### Q1: 提示"时间范围不重叠"怎么办？

A: 检查以下几点：
1. 确认视频和轨迹数据是同一天
2. 检查是否有缺失的视频文件
3. 验证轨迹数据文件是否正确

### Q2: OCR识别不准确怎么办？

A:
1. 默认情况下，脚本使用文件名时间戳，不需要OCR
2. 如果文件名没有时间戳，可以尝试：
   - 调整OCR参数
   - 使用更清晰的视频源
   - 手动标注时间范围

### Q3: 如何处理时间漂移？

A: 使用 `--time-tolerance` 参数：
```bash
python3 scripts/align_video_trajectory_v2.py --time-tolerance 5
```
